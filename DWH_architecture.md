# Intermediate Layer Design

## Purpose

The intermediate layer is responsible for transforming event-level staging data into business entities that can be reused by dimensional models and fact tables.

```text
Raw
↓
Staging
↓
Intermediate
↓
Mart (Dimensions + Facts)
```

---

# int_summary_events

## Business Meaning

Represents user activity on the Glamira website.

### Grain

```text
1 row = 1 user event
```

### Example Events

* View Product
* Search Product
* Recommendation Click
* Add To Cart
* Checkout

### Purpose

Acts as the main event dataset used to build the fact table.

### Downstream Consumer

```text
fact_user_event_detail
```

---

# int_customers

## Business Meaning

Represents unique customers identified in the event tracking system.

### Grain

```text
1 row = 1 customer
```

### Key Attributes

* customer_id
* email_address

### Purpose

Removes duplicated customer records generated by multiple website interactions.

### Example

Event data:

| event_id | customer_id |
| -------- | ----------- |
| e1       | c1          |
| e2       | c1          |
| e3       | c2          |

Intermediate result:

| customer_id |
| ----------- |
| c1          |
| c2          |

### Downstream Consumer

```text
dim_customer
```


# int_traffic_sources

## Business Meaning

Represents unique traffic source combinations.

### Grain

```text
1 row = 1 traffic source combination
```

### Key Attributes

* utm_source
* utm_medium
* referrer_url
* current_url

### Purpose

Centralizes marketing attribution data.

### Business Questions

* Which traffic source drives the most users?
* Which marketing channel has the highest conversion rate?
* Which referrals generate the most purchases?

### Example

| utm_source | utm_medium |
| ---------- | ---------- |
| google     | cpc        |
| facebook   | social     |
| newsletter | email      |

### Downstream Consumer

```text
dim_traffic_source
```

# Intermediate Layer Lineage

```text
stg_summary
    │
    ├── int_customers
    │        │
    │        └── dim_customer
    │
    ├── int_products
    │        │
    │        └── dim_product
    │
    ├── int_devices
    │        │
    │        └── dim_device
    │
    ├── int_traffic_sources
    │        │
    │        └── dim_traffic_source
    │
    └── int_summary_events
             │
             └── fact_user_event_detail
```

## Summary

| Table               | Business Entity |
| ------------------- | --------------- |
| int_summary_events  | User Activity   |
| int_customers       | Customer        |
| int_products        | Product         |
| int_devices         | Device          |
| int_traffic_sources | Traffic Source  |

The intermediate layer separates business entities from raw event tracking data and prepares them for dimensional modeling in the mart layer.
